Direction H with real drag, step, and morph. Gate label moved down into a chip that hugs the handle so it never collides with the corner pills, and clamps inside the plot at the edges. Tested on mouse and touch; responsive down to ~320 px wide.

Try it

  • Drag the coral handle — or tap anywhere on the canvas — to set Gate.
  • Drag the yellow thumb on the slider, or tap a family name, to change Shape.
  • Tap − / + on the corner pills to step Slices (2–32) and Speed (0.50×–2.00×).
  • Drag Gate all the way to each edge — the gate chip pins in place while the handle keeps moving.

Edge cases exercised

Gate label collision
Label moved down and pinned to the handle; it can never reach the top corners.
Gate at x = 0 / x = 1
Handle and dashed guide go all the way; chip clamps so it stays fully visible inside the plot.
Gate = 0 %
Envelope collapses to silence, fill and line disappear, gated region covers the plot.
Shape = Hard Gate
Envelope is a square pulse; fill has vertical sides at x = 0 and x = gate.
Bell → Ramp↓ morph
The bell's peak slides from center toward the left edge; as it pins, the Hann curvature flattens into a straight descending line. Every intermediate is a real, plausible envelope — no bell-plus-ramp hybrids.
Ramp↓ → Ramp↑ morph
Starts as a straight descending line, tightens into a plucky (1 − u)^k decay as the slider approaches the midpoint, flips direction exactly at 0.5 into a matching u^k plucky rise, then relaxes back to a straight ascending line at Ramp↑. Peak amplitude stays at 1 throughout — both sides of the flip share the same exponent k_max = 5.
Shape = Ramp↓ / Ramp↑
Envelope reaches max at one edge of the audible region and may visually brush the nearby corner pill — noted, not dodged.
Narrow viewport (≈ 320 – 480 px)
Pill buttons shrink to 20 px; below 360 px the “Slices / Speed” labels hide to keep tap targets honest.
Touch input
Pointer events + setPointerCapture, so drag continues even if the finger leaves the element.
